Answers

What is UEFI PXE booting?

UEFI PXE booting is an environment for booting computers using a network card without using hard drives, CDs, and other devices used to boot the operating system.

How does it work?

  1. Loading the PXE bootloader. The PXE firmware built into the network card loads the pxelinux bootloader.
  2. Getting the IP address and the address of the TFTP server. The pxelinux loader makes a DHCP request and uses it to get the IP address and the address of the TFTP server.
  3. The pxelinux loader accesses the TFTP server and requests the Linux kernel from it.
  4. Starting the Linux kernel. After the Linux kernel is loaded, control is transferred to it.
  5. The Linux kernel makes a DHCP request and gets its IP address, the address of the NFS server.
  6. Mounting the root file system.
  7. Calling the init process to which control is passed.

What is GPT?

GPT (GUID Partition Table) is a standard layout of partition tables of a physical computer storage device.GPT has no limitations on the size of the disk, but the system needed to be based on UEFI

What is its layout? Explain each element?

Figure 10 shows the layout of a disk with the GPT.

Figure 10 - The layout of a disk with the GPT (Source: Wikipedia)

The primary GPT Header is the first GPT data structure. It starts with the signature (“EFI PART”), the revision (00h 00h 01h 00h), and then the header size information and the CRC32 of the header. It indicates the usable logical blocks on the disk. Also, it indicates the number and size records of the partitions, forming the partition table. Figure 11 shows the GPT Header Format.

Figure 11 - The GPT Header Format

After the header, there is an array of partition entries (128 items). The first 16 bytes are used to determine the type of the GUID partition. The next 16 bytes contain a GUID that is unique to the partition. After that, an entry appears about the beginning and end of the 64-bit LBA. The rest of the space is devoted to the names and attributes of the partition.

Then there are the GPT partitions. The data located on the partitions start with the LBA 34 block.

After that, GPT has a copy of the partition entries and the partition table header. Any modifications to the GPT will change the checksum, after which the EFI will rewrite the primary GPT with the secondary one. If both of the GPTs hold an invalid checksum, access to the disk will become impossible. Therefore, this is necessary for redundancy purposes.

What is the role of a partition table?

A partition table is a data structure that provides basic information for the OS about the division of the disk into primary partitions.

What is gdisk?

gdisk is an analog of the fdisk utility but for GPT. It is a command-line utility that provides disk partitioning functions, preparatory to defining file systems.

How does it work?

According to man gdisk, when running, gdisk tries to determine the type of partition used on the disk. If it finds valid GPT data, the disk will use it. Otherwise, if it finds a valid MBR or BSD label, but without GPT data, it will try to convert the MBR or disk label to GPT form.

What can you do with it?

I can use gdisk for:

  1. creating a backup copy of GPT data;
  2. changing a partition;
  3. adding a new partition or deleting it;
  4. displaying information about partitions;
  5. creating a new GPT.

What is a Protective MBR and why is it in the GPT?

A protective MBR is a simple MBR. This provides backward compatibility for GPT with MBR. Thus, MBR-based disk utilities cannot damage GPT disks.

Figure 12 - Dump of Protective MBR

There are 446 bytes (from 0x0000 to 0x01be) of the bootstrap code area. On BIOS systems it usually contains the first stage of the boot loader. Then there are 4 partition table entries:

  1. 0000 0200 EEFF FFFF 0100 0000 2F60 383A;
    00 (0) is the status of the physical drive. It shows whether it is possible to boot the OS from this partition. 0 means inactive, 0x80 (128) - active;
    000200 is CHS (Cylinder-head-sector) address of first absolute sector in partition. Head = 0, Sector = 2, Cylinder = 0;
    ee (GPT) is partition type;
    ffffff is CHS address of last absolute sector in the partition. Head = 255, Sector = 63, Cylinder = 1023;
    01000000 (1) is LBA of first absolute sector in the partition;
    2f60383a (976773167) is the number of sectors in the partition.
  2. 0000 0000 0000 0000 0000 0000 0000 0000;
  3. 0000 0000 0000 0000 0000 0000 0000 0000;
  4. 0000 0000 0000 0000 0000 0000 0000 0000.

55aa is boot signature.

Answers

At what byte index from the start of the disk do the real partition table entries start?

Partition table entries start with an index of 0x400 (1024).
The partition starts with a byte index = first LBA (2048) * 512 = 1048576 for the first partition in my example). The first partition starts from LBA 2048, but LBA 34 is the first usable LBA as indicated in GPT.

At what byte index would the partition table start if your server had a so-called “4K native” (4Kn) disk?

The partition table will start with the third block (8192) and will have a size of 4 blocks.

If you wanted to add a 13 GiB FreeBSD ZFS partition, called ØS3 (U+00D8U+015A U+0033) to the table by hand, what values would you have to use for the entry (including the name) in the raw table on disk? Assume the disk is large enough to hold the extra partition after which you already described.

The GUID of the partition type will be ba7c 6e51 cf6e d611 8ff8 0002 2d09 712b (FreeBSD ZFS partition).
Then there is the unique partition GUID (for example - c811 e2d8 e4c3 a74f bf12 8e1c 4f3f ccec).

If I added this partition to my disk, its first LBA would be 976771072 (0058 383A 0000 0000) and the last LBA - 1002158079 (FF B7 BB 3B).

Then there is attribute flags 0000 0000 0000 0000 and name D800 5A01 3300 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 (ØS3 in little endian).

Name two differences between primary and logical partitions in an MBR partitioning scheme.

The primary partition is a bootable partition and it contains the operating system(s) of the computer, while a logical partition is a partition that is not bootable.

An MBR disk can have 4 primary partitions and no logical partitions, or 3 primary partitions and one extended partition. The extended partition can be divided into as many (Max - 26) logical partitions as you need.
